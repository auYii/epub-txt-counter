# EPUB 字数统计器

把 `.epub` 文件拖进网页，就能看到整本书、每一章的字数统计。所有解析都在浏览器里完成，文件不会上传到任何服务器。

## 功能

- 支持拖拽 / 点击上传，一次可处理多本 epub
- 自动读取书名、作者
- 同时给出多种统计口径（见下文"字数是怎么数的"）
- 每章字数明细（仿目录的点状引线样式）
- 一键复制统计结果文本
- 历史记录：手动保存到本地，支持按日期/字数/书名排序，JSON 导入导出备份
- 浅色 / 深色模式，记住你的选择
- 可作为 PWA 安装到桌面或手机主屏幕

## 部署到 GitHub Pages

1. 在 GitHub 新建一个仓库（公开仓库即可，免费版 Pages 只支持公开仓库）。
2. 把这 5 个文件上传到仓库根目录：
   - `index.html`
   - `manifest.json`
   - `sw.js`
   - `icon.svg`
   - `README.md`
3. 打开仓库的 **Settings → Pages**。
4. "Source" 选择 **Deploy from a branch**，分支选 `main`（或你的默认分支），目录选 `/ (root)`，保存。
5. 等 1～2 分钟，页面顶部会出现访问地址，形如：
   `https://你的用户名.github.io/仓库名/`
6. 打开这个地址即可使用。也可以在手机浏览器里打开后选择"添加到主屏幕"，像 App 一样使用。

> 也可以不用 GitHub：把这几个文件随便放在同一个文件夹里，双击 `index.html` 直接用浏览器打开即可，效果完全一样（首次打开需要联网，用于加载页面字体和解析库）。

## 字数是怎么数的

中文"字数"其实没有统一标准，不同小说网站、不同软件算出来的数字都不一样。所以这个工具把几种常见口径都列出来，你可以按自己的习惯选用：

| 指标 | 说明 |
|---|---|
| **总字数（headline 大数字）** | 不含空格的字符总数（含标点）。这是大多数小说站"字数"一栏最接近的口径 |
| 含空格字符数 | 把空格、换行也算进去的字符总数 |
| 纯汉字 | 只统计汉字（CJK 统一表意文字），不含标点 |
| 英文单词 | 连续的英文字母算一个单词（如 `don't` 算一个） |
| 数字串 | 连续的数字算一个，比如 `2024` 算一个，不是算 4 个字符 |
| 标点/符号 | 除汉字、英文字母、数字以外的字符（标点、符号等） |
| 智能字数(汉字+单词+数字) | 汉字数 + 英文单词数 + 数字串数，类似 Microsoft Word 对中英混排文档的"字数"统计方式，更适合英文占比较高的书 |

每一章的字数明细使用"不含空格字符数"这一口径，方便和总字数对应。

**一点小提示**：EPUB 里如果有目录页、封面页等被标记为 `linear="no"` 的页面，会被自动跳过，不计入字数，因为它们通常不是正文。

## 隐私说明

整个统计过程都在你的浏览器本地完成（用到了 [JSZip](https://stuk.github.io/jszip/) 库来解压 epub）。唯一需要联网的地方，是首次打开页面时加载 Google Fonts 字体和 JSZip 库这两个外部资源；epub 文件本身始终只停留在你的设备上，不会被发送到任何地方。历史记录保存在浏览器的 `localStorage` 里，换设备或清浏览器数据会丢失，记得用"导出 JSON"按钮定期备份。
